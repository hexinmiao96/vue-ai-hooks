# Support

Use the right channel so maintainers can respond without losing important
context.

## Questions and usage help

For setup questions, API usage, provider configuration, or integration advice,
open a GitHub Discussion when Discussions are available for the repository.

If Discussions are not available, open an issue only when you can provide enough
context for maintainers to reproduce or answer the question:

- `vue-ai-hooks` version
- Vue version
- Node and package manager versions
- Provider or custom provider adapter
- Minimal code sample or reproduction

## Bugs

Use the bug report template for reproducible defects. Include the smallest code
sample that shows the problem and the exact error message or unexpected output.

## Feature requests

Use the feature request template for new APIs, providers, or behavior changes.
Describe the problem first, then the proposed API shape.

## Security

Do not open public issues for suspected vulnerabilities, leaked provider keys,
or credential-handling problems. Follow [`SECURITY.md`](./SECURITY.md) instead.
