# Code of Conduct

## Our Standards

This project expects contributors to keep discussions practical, respectful, and
focused on improving `vue-ai-hooks`.

Examples of constructive behavior:

- Explain technical disagreements with evidence and concrete tradeoffs.
- Keep issue reports reproducible and pull request feedback actionable.
- Assume good intent, but correct unsafe or misleading guidance quickly.

Unacceptable behavior includes harassment, personal attacks, discriminatory
language, deliberate disruption, or publishing another person's private
information without permission.

## Scope

This code of conduct applies to project spaces, including issues, pull requests,
discussions, documentation, examples, and release communication.

## Reporting

Report conduct concerns to the maintainer listed in `package.json`. If the
concern involves a security issue or exposed provider credential, follow
`SECURITY.md` instead of opening a public issue.

## Enforcement

Maintainers may edit or remove harmful content, close discussions, restrict
participation, or block users when behavior creates risk for contributors or
users of the package.

Enforcement decisions should be proportionate, documented when practical, and
focused on keeping the project safe and useful.
